# دليل تشغيل ونشر التطبيق باستخدام Docker (Deployment Guide)

تم إعداد ملفات Docker و Docker Compose كاملة لتشغيل المشروع في بيئة الإنتاج بأعلى درجات الأمان والأداء. يتبع المشروع بنية ميكروسيرفس معزولة داخلياً، حيث يعمل خادم Nginx كبوابة وحيدة للمرور (Reverse Proxy) لخدمة واجهة الويب والـ API.

---

## 1. المتطلبات الأساسية (Prerequisites)
تأكد من تثبيت الأدوات التالية على السيرفر:
- **Docker** (الإصدار 20.10 أو أحدث)
- **Docker Compose** (الإصدار 2.0 أو أحدث)

---

## 2. إعداد ملف البيئة على السيرفر (Environment Configuration)
قم بإنشاء ملف باسم `.env` في المجلد الرئيسي للمشروع على السيرفر وضبط المتغيرات التالية:

```env
# المفتاح السري لتشفير الرموز (جلسات الدخول) - قم بتغييره فوراً بقيمة عشوائية قوية
SECRET_KEY=yoursupersecretkeychangeitinproduction123!

# مفتاح OpenAI لتوليد التقارير الطبية (تأكد من استخدام مفتاح صالح يبدأ بـ sk-)
OPENAI_API_KEY=sk-proj-YOUR_ACTUAL_OPENAI_API_KEY
```

---

## 3. خطوات التشغيل للمرة الأولى (Initial Deployment)

### الخطوة 1: بناء وتشغيل الحاويات (Build & Start Containers)
قم بتشغيل الأمر التالي لبناء صور الـ Docker وتشغيل الخدمات في الخلفية:
```bash
docker compose up --build -d
```
سيقوم هذا الأمر بـ:
1. إنشاء قاعدة بيانات PostgreSQL معزولة داخلياً.
2. بناء الباك اند، وإجراء الـ Migrations الجاهزة تلقائياً للتحول لقاعدة البيانات الجديدة.
3. بناء الفرونت اند (React App) بواسطة Node وحزم الملفات، ثم تشغيل خادم Nginx لاستضافتها وتمرير طلبات `/api` للباك اند.

---

### الخطوة 2: تهيئة قاعدة البيانات والبيانات الأولية (Database Seeding)
بما أنها المرة الأولى للتشغيل، يجب حقن البيانات الطبية الأولية والمستخدمين (الأدمن) في قاعدة البيانات الجديدة.

1. **لتهيئة بيانات الموقع الأساسية (الأقسام، الأطباء، الباقات، المدونات، الحسابات):**
   *(تنبيه: هذا الأمر يقوم بمسح الجداول أولاً وحقن البيانات النموذجية)*
   ```bash
   docker exec -it medical_backend python seed.py
   ```
   *ملاحظة: اسم الأدمن الافتراضي بعد هذا الأمر سيكون `admin@clinic.com` بكلمة مرور `ad1212`.*

2. **لإنشاء أو تحديث حساب الأدمن الأساسي فقط دون مسح بقية الجداول:**
   ```bash
   docker exec -it medical_backend python seed_admin.py
   ```
   *ملاحظة: هذا الأمر ينشئ حساب أدمن `admin@clinic.com` بكلمة مرور `admin123`.*

---

## 4. إدارة الخدمات (Service Management)

- **لعرض حالة الحاويات والتأكد من عملها:**
  ```bash
  docker compose ps
  ```

- **لمراقبة سجلات التشغيل (Logs) ومتابعة الأخطاء:**
  ```bash
  docker compose logs -f
  ```

- **لإيقاف الخدمات بالكامل دون حذف البيانات:**
  ```bash
  docker compose down
  ```

- **لتحديث الكود البرمجي وإعادة التشغيل (عند إجراء تعديلات جديدة):**
  ```bash
  docker compose up --build -d
  ```

---

## 5. الأمان وتأمين الخادم (Production Security Notes)
- **المنافذ المفتوحة (Open Ports):** تم إعداد الفرونت اند (Nginx) ليعمل على المنفذ `3000` والباك اند ليعمل على المنفذ `8000` وتصديرهما للخارج في ملف `docker-compose.yml`.
- **التشغيل خلف HTTPS (موصى به جداً للإنتاج):**
  يفضل تشغيل خادم Nginx الخارجي على السيرفر لربط شهادة SSL مجانية (Let's Encrypt / Certbot) وتوجيهها للمنفذ `3000` الداخلي للـ Docker.
